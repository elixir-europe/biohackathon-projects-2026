# Project 25: DBTLHUB Interoperability Contracts: Standardised Phase Transitions for Engineering Biology

## Abstract

DBTL workflows in engineering biology remain fragmented at phase transition points. Tools for design, build, test, and learn excel within their domains, but the exchange of outputs, metadata, provenance, and quality information between phases is inconsistent, often manual, and poorly standardised. This creates unnecessary friction for researchers trying to connect existing resources into coherent workflows.
 
 This project defines a lightweight interoperability contract for DBTL phase transitions, delivering concrete schemas for Design→Build (construct specifications), Build→Test (assembly outputs), Test→Learn (phenotypic screening data), and Learn→Design (model predictions). Each schema specifies minimum artefacts, metadata requirements, provenance tracking, and quality signals needed for reliable handovers.
 
 We will implement a minimal OpenAPI service and reference output wrappers/adapters specifications, demonstrating how tool outputs convert to standardised handover packages. A demonstrator workflow will validate end-to-end exchange across at least two transitions.
 
 The approach makes phase handovers machine-readable and reusable without requiring tools to be rewritten. Outputs integrate naturally with ELIXIR services (bio.tools, Galaxy) and provide a practical foundation for DBTLHUB's one-stop resource hub.
 
 The result is a reusable specification that reduces manual reformatting, improves reproducibility, and clarifies how heterogeneous DBTL tools connect into working workflows, establishing a concrete interoperability layer for engineering biology that scales with community adoption.
