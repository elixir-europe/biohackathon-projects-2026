# Project 12: SampleSift: spatial querying and selection of geolocation-tagged biological samples

## Abstract

Multiple ELIXIR Core Data Resources (CDRs), including ENA and MGnify, hold vast collections of biological samples with geographic coordinates, driven by exponential data growth and enforced metadata requirements. Yet spatial exploration remains limited: existing tools offer only basic geographic query builders, requiring further code to combine them with advanced metadata filtering or prepare intuitive data visualisations. These limitations extend to myriad resources both within and outside ELIXIR communities, for example where relevant data are deposited to generic data repositories like Zenodo, or in bespoke data portals. Emerging data resources built on-top of existing reusable codebases and infrastructures are destined to inherit these limitations.
 
 SampleSift will address this gap with a modular, open-source stack of technologies for spatially querying and filtering large biological sample collections. A database layer of spatial-specific tooling (e.g. PostGIS for Postgres, SpatiaLite for SQLite) is paired with an interactive Leaflet.js map interface, allowing users to draw geographic regions of interest and retrieve matching samples in real time, filtered simultaneously by taxonomy, collection date, environment biome, and other metadata dimensions.
 
 The architecture is explicitly designed for scale (e.g. 100,000+ records) and for reuse: the embeddable map widget, RESTful faceted search API, and spatial query layer can each be adopted independently by any web data portal. We intend for MGnify, ELIXIR’s CDR for microbiome-derived data, to be an exemplar use case. This hackathon will deliver a live prototype on real CDR data, a reusable map widget, an OpenAPI-documented faceted search API, and containerised deployment documentation.
