# Project 5: A submission and visualisation interface for sharing community interoperability practices

## Abstract

Interoperability practices across stakeholder groups, whether through projects, institutes, platforms or communities, are often shared as informal know-how in slides, wikis, or email threads, making them hard to find, compare, improve and reuse. Building on an ongoing ELIXIR Interoperability Platform (EIP) effort to collect practices using the RDMkit Data Life Cycle framing, this project will develop a lightweight submission and visualisation interface to share reference community practices across the research data management ecosystem.
 
 The project builds on an existing questionnaire, currently undertaken as a “guided” interview, that already captures the key elements to enable the “FAIR” re-execution of a “task”, including domain or context, lifecycle stage, tools and resources used, and key constraints. We will turn this into a guided workflow that allows stakeholders (ELIXIR members, community champions, data stewards, etc) to submit practices directly, without needing an interview. Submitted practices will be exportable in machine-readable formats such as JSON and YAML, and displayed in human-friendly views such as data lifecycle maps, comparison tables, and tool-assembly-style diagrams. The prototype will include examples from several ELIXIR communities and link to relevant resources such as RDMkit, the FAIR Cookbook, FAIRsharing, bio.tools, github, etc.
 
 By making it easier for communities to contribute, find and reuse interoperability-enhanced practices, the interface will help them identify shared solutions, spot gaps, and move more quickly toward interoperable workflows, tools, and metadata approaches.
