# Project 20: Automated Software Maturity Assessment and Tool Disambiguation in bio.tools

## Abstract

This project aims to enrich ELIXIR’s bio.tools software registry by applying a new, automated, research software maturity model to the approximately 10,000 tools in bio.tools that are linked to a GitHub, GitLab or Codeberg repository, and are described in at least one publication. The project will evaluate whether automated maturity (e.g., documentation quality, FAIR compliance, maintenance, and reuse signals) assessment can provide a scalable and informative complement to manual maturity annotation and assign up-to-date maturity levels.
 
 As a long-term goal, the maturity model will become part of software disambiguation that goes beyond names, descriptions and topics, allowing for more fine-grained searchability among tools with similar functionality, building on the wealth of metadata in the Research Software Ecosystem to assist researchers searching for and making informed selections of software tools.
 
 The project combines software metadata from bio.tools and other services of the Research Software Ecosystem, including OpenEBench, repository metadata from code hosting platforms, publication metadata from Europe PMC, and static code analysis using open-source software adapting and building on an automated maturity model recently developed in the more narrowly scoped TDCC-LSH “FAIR Toolbox” project in the Netherlands. The expected outputs are a benchmarked workflow to automatically retrieve relevant metrics from the available information, a first maturity assessment for a large subset of bio.tools, and recommendations for how maturity can be integrated into bio.tools curation and display workflows.
