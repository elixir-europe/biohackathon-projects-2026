# Project 24: Evaluating and Securing AI-Assisted Workflows for Sensitive Data

## Abstract

Handling sensitive data in cloud environments presents a significant challenge: researchers need flexible analysis tools while data custodians must ensure privacy and prevent data exfiltration. Lately, coding agents became popular and are being used for many tasks, including workflow creation. The idea of automatic code generation for sensitive data processing evokes immediate security concerns. Workflow execution in a secure environment must fulfill certain safety standards and requires deterministic behavior. This project aims to deliver best practices and safety guidelines regarding code generating AI tools in Secure Processing Environments. As part of the process, a pilot platform will be created to find vulnerabilities, develop countermeasures and validate them.
 
 In this proposed platform, an agent system is used for code generation and for monitoring data integrity. Users interact with an agent that has access to necessary tools and metadata, and constructs a Snakemake workflow based on the user's needs. To ensure reproducibility and security, the workflow is executed deterministically via WESkit, running independently of the agent. Results will be documented using RO-crates. A separate security validation agent, augmented with deterministic static code analysis tools, reviews the code to prevent harmful code execution. It applies standard Data Loss Prevention filters before sharing them with the user. Ideally, the steps required to ensure secure workflow execution are formalized in such a way that they can be followed autonomously.
 
 The goal of this BioHackathon project is to prototype this platform, actively conduct security testing to identify potential vulnerabilities and develop best practices for AI-driven SPEs.
