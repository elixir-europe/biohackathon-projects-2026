# Project 31: Nextflow to Galaxy Workflow Portability and FAIRness via Modular LLM Agent Skills

## Abstract

Computational workflows encode domain expertise and operating procedures, and large, curated catalogues of production pipelines have emerged, such as nf-core for Nextflow and the Intergalactic Workflow Commission (IWC) for Galaxy. However, the reimplementation of these assets across languages remains a mostly-manual, resource-heavy work, and thus a bottleneck for FAIR, reproducible science. To improve workflow portability between Nextflow and Galaxy, we propose a "maturation ladder", based on Large Language Model (LLM) agent skills:
 
  * Translation: A Nextflow to Galaxy porting skill (via CWL), extending recently developed work .
 
  * Prototyping: On-the-fly user-defined tools (UDTs) that wrap missing tools or custom nf-core scripts, making ports immediately end-to-end runnable.
 
  * Hardening: A tool-authoring skill that promotes UDTs into standard, best-practice Galaxy tools ready for community review, providing reusable wrappers rather than throwaway glue.
 
  * Maintenance: A sync mechanism that keeps ported pipelines and wrapped tools in lockstep with upstream nf-core releases.
 
 To ensure high standards, we will extend the existing Galaxy workflow Best Practice panel with an LLM-based FAIRness evaluator. While the current panel checks for basic metadata, our extension will leverage LLMs to evaluate deeper criteria from established community principles (e.g. 10.1038/s41597-025-04451-9 ), providing actionable feedback for both new ports and existing workflows.
 
 These capabilities will be exposed via a Model Context Protocol (MCP) server for any agent or CLI, and as a ChatGXY plugin inside Galaxy's web interface. This modular architecture decomposes the complex translation task into smaller, parallelisable components, allowing diverse teams to contribute simultaneously towards a more unified and FAIR bioinformatics ecosystem.
